# t_email.md  
Email Template — Table Version

| Section | Name | Description |
|---------|------|-------------|
| **1** | Subject Line | A clear, concise subject that signals the purpose of the email. |
| 1.1 | Purpose Signal | Indicates exactly why you are writing (request, update, follow-up). |
| 1.2 | Keywords | Important searchable terms placed early in the subject. |
| **2** | Greeting | Sets tone based on formality and relationship. |
| 2.1 | Formal Greeting | Used for academics, executives, or formal communication. |
| 2.2 | Informal Greeting | Used for peers, teams, colleagues, or familiar contacts. |
| **3** | Opening Line | Immediately provides context so the reader understands the purpose fast. |
| 3.1 | Relationship Recall | Brief reminder of who you are or why you're contacting them again. |
| 3.2 | Immediate Purpose | States directly what the email is about. |
| **4** | Core Message | Main content: request, update, explanation, or clarification. |
| 4.1 | Request or Update | The key action or information being delivered. |
| 4.2 | Context | Background details needed to understand the request/update. |
| 4.3 | Key Points | Bulleted essentials if multiple items must be conveyed. |
| **5** | Call to Action | What you want the recipient to do next. |
| 5.1 | Specific Action | A clear, actionable instruction. |
| 5.2 | Deadline (Optional) | When the action is needed by. |
| **6** | Closing | A respectful, professional end to the message. |
| 6.1 | Standard Close | “Thank you,” “Best regards,” and similar endings. |
| 6.2 | Warm Close | “Appreciate your help,” “Thanks again,” etc. |
| **7** | Signature | Professional identity and contact block. |
| 7.1 | Contact Details | Email, phone, role, organization. |
| 7.2 | Links | Optional LinkedIn, website, or portfolio links. |
| **8** | Optional Elements | Supplemental components that support or enhance the message. |
| 8.1 | Attachments | Files referenced explicitly in the email body. |
| 8.2 | Postscript | Optional P.S. for urgent or secondary notes. |
